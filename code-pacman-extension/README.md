# Code Pacman

把当前 VS Code 编辑器临时变成一个“摸鱼吃代码”小游戏。

## 玩法

- `Ctrl+D`: 开始游戏 / 隐藏游戏并恢复代码 / 再次继续
- `Alt+↑ / ↓ / ← / →`: 控制小人移动
- `Esc`: 退出游戏并恢复代码

游戏会读取当前打开文件的文本内容，把它当成地图。小人走过非空白字符时，就像把代码“吃掉”一样，那些字符会临时消失。

## 设计说明

- 这是一个快照式原型：启动时会记住当前文件内容。
- 隐藏游戏时，会把原始代码立即写回编辑器，所以看上去就像什么都没发生。
- 再次按 `Ctrl+D` 时，如果你在隐藏期间改过代码，关卡会按新代码重新生成。

## 注意

- 游戏显示期间不要保存文件。
- 更稳妥的体验方式是：先保存代码，再开玩。
- 当前原型只对“当前编辑器”生效，不会识别 VS Code 的整张屏幕截图，而是直接操作真实代码文本，这样恢复最稳定。

## 本地调试

1. 用 VS Code 打开这个文件夹。
2. 按 `F5` 启动 Extension Development Host。
3. 在新窗口里打开任意代码文件。
4. 按 `Ctrl+D` 开始。

## 安装方式

### 方式一：本地安装 VSIX

把项目打包成 `.vsix` 后，可以在 VS Code 里通过“Extensions: Install from VSIX...”直接安装。

### 方式二：发布到 VS Code Marketplace

发布到 Marketplace 后，别人就可以直接在 VS Code 扩展面板里搜索并下载安装。

目前这个仓库已经是标准扩展结构，离正式发布还差两步：

1. 注册一个 VS Code Marketplace publisher。
2. 用 publisher 身份打包并发布当前扩展。
